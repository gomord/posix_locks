            Charles Strahan Dilan Manatunga • 8 months ago

            The linker can't know ahead of time where another shared library will be loaded in memory. Remember that this is about inter-shared library calls, not intra-shared library calls. However, at load time the dynamic loader knows the offsets of any two shared libraries (it loaded them, after all), and it then needs a way to hand the offsets to the program; it does this by storing the offsets in the GOT.
